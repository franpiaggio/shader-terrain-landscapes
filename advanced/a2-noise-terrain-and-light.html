Noise terrain & light

A1's terrain was already fbm noise — but flat-shaded, it reads as a paper cutout. Give it a normal and a sun, and the same height map suddenly has form: ridges catch light, valleys fall into shadow.

Your win this lesson Compute the terrain normal from the height map and light it with diffuse + sky ambient — turning the silhouette into a 3-D landform.

Normals from a height map

You don't need the SDF tetrahedron trick here. The surface is y = h(x,z), so the normal comes straight from how the height changes in x and z (finite differences):

vec3 calcNormal(vec3 p, float t) {
    float e = 0.002 * t;            // scale the sample gap with distance (anti-alias)
    return normalize(vec3(
        terrain(p.xz - vec2(e,0)) - terrain(p.xz + vec2(e,0)),
        2.0 * e,                     // the "up" component
        terrain(p.xz - vec2(0,e)) - terrain(p.xz + vec2(0,e))));
}

Steep slopes → big height differences → normal tilts away from up; flat ground → normal points up. Then it's the same Lambert lighting as the core course: diffuse = max(dot(n, sunDir), 0.0), plus a sky-blue ambient so shadowed slopes aren't black.

Play — move the sun

Single-material terrain, fully lit. Drag sun x / y to move the light and watch ridges and valleys appear — that's the normal doing its job. The camera flies forward.

Now you — light it, graded live

The terrain is painted a flat green — no form. Compute the normal and apply diffuse + ambient. Check verifies the surface now varies (lit ridges vs shaded slopes) instead of being uniform.

Watch for The sample gap e must scale with distance (0.002*t). A fixed tiny e makes far normals noisy (aliasing); too large smears the detail. And keep some ambient — pure diffuse makes shadowed slopes read as holes.
