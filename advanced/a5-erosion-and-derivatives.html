Noise derivatives & erosion

Plain fBM gives you bubbly hills — every bump the same blobby size, no ridgelines, no carved valleys. One idea closes most of the gap to a real mountain: let each octave of noise know its own slope, and damp the fine detail where the ground is already steep. That single trick is erosion.

Your win this lesson Swap the blobby height map for an eroded one built from noise derivatives — turning rolling lumps into sharp ridges and smooth, scooped valleys.

Why plain fBM looks fake

Standard fBM just sums octaves of noise: a += b*noise(p). Every octave is added at full strength everywhere, so detail piles onto detail uniformly — you get a field of similar-sized bumps. Real terrain isn't like that: water and gravity carry material downhill, so steep slopes stay smooth (detail washed away) while flats and ridgelines keep their texture. To fake that, the height function needs to know how steep it already is at each octave.

Noise that knows its slope

Ordinary noise() returns just a height. noised() (IQ's "morenoise") returns the height and its two partial derivatives — the slope in x and z — in one vec3. It uses a quintic curve so those derivatives are smooth:

vec3 noised(vec2 x) {            // returns vec3(height, d/dx, d/dz)
    vec2 i = floor(x), f = fract(x);
    vec2 u  = f*f*f*(f*(f*6.0-15.0)+10.0);    // quintic smootherstep
    vec2 du = 30.0*f*f*(f*(f-2.0)+1.0);        // its derivative
    float a=hash(i), b=hash(i+vec2(1,0)), c=hash(i+vec2(0,1)), d=hash(i+vec2(1,1));
    return vec3(a + (b-a)*u.x + (c-a)*u.y + (a-b-c+d)*u.x*u.y,
                du * (vec2(b-a, c-a) + (a-b-c+d)*u.yx));
}

Erosion: damp detail on steep ground

Now the trick. As you sum octaves, also accumulate the slope in d. Divide each new octave by 1 + dot(d,d) — so where the running slope is large (steep), fine detail is suppressed; where it's flat, detail comes through. A few lines turn bubbles into landforms:

float fbmEroded(vec2 p) {
    float a = 0.0, b = 1.0;
    vec2 d = vec2(0.0);                      // accumulated slope
    for (int i = 0; i < 8; i++) {
        vec3 n = noised(p);
        d += n.yz;                            // add this octave's slope
        a += b * n.x / (1.0 + dot(d,d));      // damp it where it's already steep
        b *= 0.5;  p = m2 * p * 2.0;          // next octave: half amplitude, double freq
    }
    return a;
}
Set erode to 0 and it's the old fBM If you scale the accumulated slope by an erode knob (d += erode*n.yz), then erode = 0 makes dot(d,d) = 0 and the formula collapses back to a plain a += b*n.x sum — the exact bubbly fBM from A2–A4. Now you — write the erosion, graded live

The height function loops over octaves but never adds them, so the terrain is flat. Add the two lines that accumulate the slope and add each octave damped by it. Check verifies real ridges and valleys appear (the surface stops being uniform).

Watch for The derivatives n.yz must be accumulated before you use dot(d,d) in the same octave, or the first octave isn't damped. And erosion makes slopes steeper than plain fBM — if thin ridges look eaten away, the heightfield march can overshoot them; take slightly smaller steps (dt) or raise the iteration count.
